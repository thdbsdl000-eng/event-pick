# 배포 가이드

프로젝트 빌드가 성공적으로 완료되었습니다! 이제 웹사이트를 배포할 수 있습니다.

## 🚀 추천 배포 플랫폼: Vercel

Vercel은 Next.js를 만든 회사에서 제공하는 플랫폼으로, Next.js 프로젝트 배포에 가장 적합합니다.

### Vercel 배포 방법

#### 1단계: GitHub에 코드 업로드 (선택사항이지만 권장)

1. **GitHub 계정 만들기** (없는 경우)
   - https://github.com 접속
   - 회원가입

2. **새 저장소 만들기**
   - GitHub에서 "New repository" 클릭
   - 저장소 이름 입력 (예: `event-banner-ai`)
   - "Create repository" 클릭

3. **코드 업로드**
   - GitHub에서 제공하는 명령어를 따라하거나
   - GitHub Desktop 앱 사용 (더 쉬움)

#### 2단계: Vercel에 배포

1. **Vercel 계정 만들기**
   - https://vercel.com 접속
   - "Sign Up" 클릭
   - GitHub 계정으로 로그인 (권장)

2. **프로젝트 배포**
   - Vercel 대시보드에서 "Add New Project" 클릭
   - GitHub 저장소 선택 (또는 "Import Git Repository")
   - 프로젝트 설정:
     - **Framework Preset**: Next.js (자동 감지됨)
     - **Root Directory**: `./` (기본값)
     - **Build Command**: `npm run build` (자동 설정됨)
     - **Output Directory**: `.next` (자동 설정됨)

3. **환경 변수 설정** (중요!)
   - "Environment Variables" 섹션으로 이동
   - 다음 변수 추가:
     - **Name**: `GEMINI_API_KEY`
     - **Value**: Google Gemini API 키 (`.env.local`에 있는 값)
   - "Add" 클릭

4. **배포 시작**
   - "Deploy" 버튼 클릭
   - 몇 분 후 배포 완료!

5. **배포 완료**
   - 배포가 완료되면 자동으로 URL이 생성됩니다
   - 예: `https://event-banner-ai.vercel.app`
   - 이 URL로 사이트에 접속할 수 있습니다!

## 🔧 다른 배포 방법

### Netlify 배포

1. https://netlify.com 접속
2. "Add new site" → "Import an existing project"
3. GitHub 저장소 연결
4. 빌드 설정:
   - Build command: `npm run build`
   - Publish directory: `.next`
5. 환경 변수에 `GEMINI_API_KEY` 추가
6. "Deploy site" 클릭

### 자체 서버 배포

1. 서버에 Node.js 설치
2. 프로젝트 파일 업로드
3. `npm install` 실행
4. `.env.local` 파일에 `GEMINI_API_KEY` 설정
5. `npm run build` 실행
6. `npm start` 실행
7. 포트 3000으로 접속

- [ ] `.env.local` 파일에 `GEMINI_API_KEY`가 설정되어 있는지 확인
- [ ] 빌드가 성공적으로 완료되었는지 확인 (`npm run build`)

## 📝 환경 변수 설정 방법

### Vercel에서 환경 변수 설정

1. Vercel 대시보드 → 프로젝트 선택
2. "Settings" → "Environment Variables"
3. "Add New" 클릭
4. Name: `GEMINI_API_KEY`
5. Value: API 키 입력
6. Environment: Production, Preview, Development 모두 선택
7. "Save" 클릭

### Netlify에서 환경 변수 설정

1. Netlify 대시보드 → 프로젝트 선택
2. "Site settings" → "Environment variables"
3. "Add variable" 클릭
4. Key: `GEMINI_API_KEY`
5. Value: API 키 입력
6. "Save" 클릭

배포가 완료되면:

- 전 세계 어디서나 접속 가능합니다
- 자동으로 도메인 이름이 생성됩니다 (예: `your-project.vercel.app`)
- 커스텀 도메인도 연결할 수 있습니다

## 💡 문제 해결

### 빌드 오류가 발생할 때
- `npm run build`를 로컬에서 실행하여 오류 확인
- 오류 메시지를 확인하고 수정

### 환경 변수가 작동하지 않을 때
- Vercel/Netlify에서 환경 변수가 올바르게 설정되었는지 확인
- 변수 이름이 정확한지 확인 (`GEMINI_API_KEY`)
- 배포를 다시 시작해보세요

### API 호출이 실패할 때
- Gemini API 키가 유효한지 확인
- API 키에 충분한 할당량이 있는지 확인
- 브라우저 콘솔에서 오류 메시지 확인
